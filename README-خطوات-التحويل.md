# رفيق التسوق — دليل التحويل إلى تطبيق أندرويد (PWA → APK)

هذه الحزمة تحتوي على نسخة "تطبيق ويب تقدمي" (PWA) كاملة من تطبيقك، جاهزة لتصبح تطبيق أندرويد
يعمل بكامل صلاحيات المتصفح الحقيقي (طباعة، حفظ PDF، مشاركة، كاميرا) داخل واجهة تبدو تطبيقًا أصليًا.

محتويات الحزمة:
- index.html          → التطبيق نفسه (نقطة الدخول)
- manifest.json       → إعدادات التطبيق (الاسم، الألوان، الأيقونات)
- service-worker.js   → يجعل التطبيق يعمل دون إنترنت بعد أول فتح
- icons/              → أيقونات التطبيق بهوية بصرية متناسقة

---

## الخطوة ١: استضافة الملفات على رابط إنترنت (مجانًا)

أسهل خيار بدون أي تسجيل: **Netlify Drop**

1. افتح: https://app.netlify.com/drop
2. اسحب مجلد الحزمة بالكامل (بكل الملفات بداخله) وأفلته في الصفحة
3. خلال ثوانٍ سيعطيك رابطًا مثل: `https://your-app-name.netlify.app`
4. افتح الرابط وتأكد أن التطبيق يعمل بشكل طبيعي من المتصفح

ملاحظة: هذا الرابط المجاني مؤقت ما لم تُنشئ حسابًا مجانيًا وتربطه بمشروع دائم من نفس الموقع
(اضغط "Sign up" لحفظ الموقع بشكل دائم مجانًا).

بدائل أخرى بنفس الفكرة: GitHub Pages، Vercel — كلها مجانية وتصلح.

---

## الخطوة ٢: تحويل الرابط إلى تطبيق أندرويد (TWA) عبر PWABuilder

1. افتح: https://www.pwabuilder.com
2. الصق رابط موقعك الذي حصلت عليه في الخطوة ١، واضغط "Start"
3. سيفحص PWABuilder الموقع تلقائيًا (Manifest, Service Worker, الأيقونات) — يفترض أن تظهر علامات ✓ خضراء
4. اضغط "Package for Stores" ثم اختر **Android**
5. اترك الإعدادات الافتراضية (أو عدّل اسم الحزمة إن أردت)، ثم اضغط "Generate"
6. سيُنزَّل ملف مضغوط يحتوي على APK أو AAB جاهز — ثبّته على جوالك مباشرة

---

## بعد التثبيت

التطبيق سيعمل الآن داخل محرك كروم الحقيقي (Trusted Web Activity)، وليس WebView مبتور الصلاحيات، لذلك:
- مسح الباركود بالكاميرا يعمل بثبات
- تصدير PDF (حراري أو A4) يُحفظ ويُشارك بشكل طبيعي تمامًا
- التطبيق يعمل دون إنترنت بعد أول فتح بفضل service-worker.js

إن واجهت أي مشكلة في خطوة معينة، أخبرني برقم الخطوة تحديدًا وسأساعدك.
